# km-page-optimizer

## 适用场景

- 页面结构太散，想统一成框架风格
- 想把原生 HTML 或自定义实现替换成 Kpu-mobile 内建组件
- 想优化弹窗、加载、空状态、表单或卡片区域的实现方式

## 使用方式

直接说明以下信息即可：

- 目标应用，例如 `example`
- 目标页面路径
- 你希望优化的方向，例如“统一 UI 风格”或“替换为内建组件”

示例：

```text
在 example 应用里优化订单详情页，
把原生 button、弹窗和空状态都替换成 Kpu-mobile 内建组件。
```

## 结果

通常会修改：

- 目标页面文件

必要时会参考：

- `apps/<app>/src/ui/components/`
- 对应组件 README 文档
